대규모 프로젝트에서 코드 버전 관리와 비교하여 데이터베이스 버전 관리 시 발생하는 주요 어려움은 무엇
-> 데이터베이스 구조 변경 시 기존 데이터가 손실 가능
데이터베이스 구조 변경은 데이터의 영속성과 직결되어 잘못 관리될 경우 기존 운영 데이터가 손상되거나 손실될 수 있다.

커스텀 ORM 방식에서 데이터베이스의 테이블, 컬럼, 프로시저와 같은 구조는 주로 어떤 형태로 정의
-> XML 파일을 이용
커스텀 ORM은 `game.db.xml` 파일에 데이터베이스 테이블 구조와 스토어드 프로시저를 정의하고 이를 파싱하여 사용

자동화된 Procedure Generator 도구의 주된 목적은 무엇
-> XML에 정의된 스토어드 프로시저 호출을 위한 C++ 래퍼 클래스를 자동 생성
Procedure Generator는 XML 정의를 읽어 스토어드 프로시저를 C++ 코드에서 편리하고 안전하게 호출할 수 있는 래퍼 코드를 자동으로 생성
